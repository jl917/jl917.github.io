#Decomposition

#1. 설명

**Decomposition(분해)**는 복합 질의를 여러 개의 단순한 하위 질의(sub-queries)로 쪼갠 뒤, 각각 검색·처리하고 결과를 합쳐서 최종 답을 만드는 Query·Reasoning 기법입니다.

"여러 조건을 동시에 만족"하거나 "여러 단계가 필요한 질문"을 작은 질문으로 나누어 검색과 추론의 정확도를 높입니다.

#2. 작동 방식

#1 복합 질의 식별

  • 하나의 질문에 여러 주제·조건·단계가 섞여 있으면 분해 후보
  • 예: "A 회사 제품 중 2023년에 리콜된 제품의 보상 정책은?" → 회사 제품 목록, 리콜 목록, 보상 정책으로 분해

#2 하위 질의 생성

  • LLM 또는 규칙으로 독립적으로 검색 가능한 sub-query 리스트 생성
  • 각 sub-query는 단일 주제·단일 조건에 가깝게

#3 각 sub-query로 검색·답변

  • 각 하위 질의에 대해 retriever로 문서 검색 후 LLM이 답변(또는 중간 결과) 생성
  • 필요 시 하위 답변을 다음 sub-query의 컨텍스트로 사용

#4 결과 통합

  • 하위 답변·검색 결과를 모아 최종 질의에 대한 하나의 답으로 통합 (LLM 합성 또는 규칙)

#3. 장점 (Benefits)

  • 복잡한 질의에 대한 검색·추론 정확도 향상
  • 각 단계별로 적절한 문서만 가져와 노이즈 감소
  • **다단계 추론(chain-of-thought)**과 결합 시 설명 가능성·디버깅 용이
  • Agent·Tool 사용과 맞닿아 있음 (질의 → 계획 → 하위 작업 실행 → 통합)

#4. 예제
