# Expansion

## 1. 설명
------

**Expansion(쿼리 확장)**은 **원래 질의에 동의어·관련어·하위 개념을 추가**하여 검색어를 넓혀 **recall을 높이는** Query Rewriting 기법입니다.

Lexical rewrite의 "키워드 추가"를 확장한 개념으로, **의미적으로 같은 표현·연관 표현**을 더해 검색 결과 누락을 줄입니다.

## 2. 작동 방식
---------

### 1 동의어·유사어 추가

*   쿼리 내 핵심어에 대한 동의어·유의어를 붙여 검색 (예: 자동차 → 차, automobile, vehicle)
*   시소러스·도메인 사전 또는 LLM으로 확장어 후보 생성

### 2 상위·하위 개념 확장

*   상위어(hypernym): "사과" → "과일"
*   하위어(hyponym): "과일" → "사과, 배, 포도"
*   도메인에 따라 상위/하위만 적용해 범위 조절

### 3 LLM 기반 확장

*   질의를 LLM에 주고 "검색에 유용한 추가 키워드/문구" 생성
*   Multi-query와 결합해 여러 확장 쿼리로 각각 검색 후 병합 가능

### 4 적용 시점

*   **인덱싱 시** 문서 쿼리 확장(문서 쪽 확장)은 드물고, 주로 **검색 시 쿼리 확장**에 사용

## 3. 장점 (Benefits)
-----------------

*   **표현 다양성**으로 인한 검색 누락 감소 (recall 향상)
*   **BM25·키워드 검색**에서 특히 효과적
*   **도메인 맞춤 확장**으로 전문 용어·일상어 매칭 개선
*   **Multi-query, HyDE** 등과 함께 사용 시 검색 품질 추가 향상
