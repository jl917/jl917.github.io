#HyDE

#1. 설명

**HyDE(Hypothetical Document Embeddings)**는 실제 문서를 검색하는 대신, 질의에 대한 "가상의 답변 문서"를 LLM으로 생성한 뒤 그 가상 문서를 임베딩하여 유사한 실제 문서를 찾는 Retrieval 기법입니다.

질의와 문서 간 **표현 차이(asymmetry)**를 줄이기 위해, 답변과 비슷한 문체·용어를 가진 가상 문서를 매개로 사용합니다.

#2. 작동 방식

#1 가상 답변 생성

  • 사용자 질의를 LLM에 넣고, 그 질문에 답하는 짧은 문단(가상 문서) 1~N개 생성
  • 실제 지식 없이 "그럴듯한 답변" 형태로 생성 (hallucination 허용)

#2 가상 문서 임베딩

  • 생성된 가상 답변을 동일한 embedding 모델로 임베딩
  • 여러 개 생성 시 각각 임베딩 후 검색해 결과 병합 가능

#3 실제 문서 검색

  • 가상 문서 임베딩을 쿼리 벡터로 사용해 벡터 DB에서 유사 문서 검색
  • 반환된 실제 문서를 RAG 컨텍스트로 사용

#3. 장점 (Benefits)

  • 질의-문서 표현 불일치 완화: 질문 문장보다 "답변 형태"가 문서와 유사해 임베딩 검색에 유리
  • 의미 공간 정렬: 답변 공간과 문서 공간이 가까워져 recall 개선
  • 키워드 검색만으로 놓치기 쉬운 의미적 유사 문서 확보에 도움
  • 다중 가상 문서 생성 + 병합 시 안정성·다양성 확보 가능

#4. 예제

질의(키워드)를 입력으로, 그 질문에 답하는 **가상의 문단(가상 문서)**을 생성한 예시입니다. 이 가상 문서를 임베딩해 유사 실제 문서를 검색합니다.
