#Injection

#1. 설명

**Injection(주입)**은 RAG·검색 파이프라인에서 특정 정보를 쿼리·프롬프트·컨텍스트에 의도적으로 넣어 검색 품질이나 답변 품질을 제어하는 기법을 통칭합니다.

쿼리 강화(키워드/문구 주입), 메타데이터·필터 조건 주입, 시스템 지시문 주입 등이 여기 해당합니다.

#2. 작동 방식

#1 쿼리 보강 주입

  • 사용자 질의 앞뒤에 도메인 키워드·동의어를 붙여 검색어 확장
  • 예: "반품 절차" → "반품 절차 정책 환불 교환" (검색어 주입)

#2 메타데이터·필터 주입

  • 쿼리와 함께 필터 조건(날짜, 카테고리, 소스)을 retriever에 전달
  • 예: "최신 정책" + date >= 2024 같이 메타데이터 주입

#3 시스템·역할 주입 (Prompt)

  • 검색 결과를 사용하는 LLM 프롬프트에 역할·형식·제약을 명시
  • "검색된 문서만 참고하여 답하라", "출처를 반드시 명시하라" 등

#4 주의사항

  • Prompt Injection 공격에 유의: 사용자 입력에 악의적 지시가 섞이지 않도록 검증·샌드박싱 필요

#3. 장점 (Benefits)

  • 검색 범위·품질 제어 (도메인·기간·소스 한정)
  • 일관된 답변 형식·정책 유도 (시스템 지시 주입)
  • 재현 가능한 파이프라인 구성에 유리
  • 보안·정책 반영 (예: 특정 문서만 참조하도록 주입)
