#Multi-Query

#1. 설명

Multi-Query는 하나의 사용자 질의에서 여러 개의 쿼리 변형을 생성하고, 각 변형으로 검색한 뒤 결과를 합쳐(merge) 사용하는 Query Rewriting·Retrieval 기법입니다.

LLM을 이용해 질의를 다른 표현·관점으로 바꾼 여러 쿼리를 만들어 단일 쿼리 검색의 한계(표현 의존성, 누락)를 보완합니다.

#2. 작동 방식

#1 다중 쿼리 생성

  • 사용자 질의를 입력으로 LLM에 여러 변형 생성 요청 (예: 3~5개)
  • 동의어 치환, 관점 변경, 구체화·일반화 등 다양하게 변형

#2 각 쿼리로 검색

  • 각 변형 쿼리에 대해 동일한 retriever로 문서/청크 검색
  • 벡터 검색 또는 BM25 등 동일 설정 유지

#3 결과 병합(Merge)

  • Reciprocal Rank Fusion (RRF) 등으로 순위 재계산
  • 중복 문서 제거 후 상위 k개 선택하여 컨텍스트로 사용

#3. 장점 (Benefits)

  • 단일 쿼리 표현에 따른 누락 감소 (recall 향상)
  • 다양한 관점의 문서를 한 번에 확보 가능
  • 구현이 비교적 단순 (기존 retriever + LLM 호출)
  • HyDE, expansion 등과 조합해 더 풍부한 검색 가능

#4. 예제
