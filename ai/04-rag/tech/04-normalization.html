#Normalization

#1. 설명

**Normalization(정규화)**은 쿼리와 문서를 일관된 형태로 맞추어 서로 다른 표기·형태가 같은 의미로 매칭되도록 하는 전처리 기법입니다.

대소문자, 공백, 유니코드, 어형 변화(stemming/lemmatization) 등을 통일해 검색 recall을 높이고 불일치로 인한 누락을 줄입니다.

#2. 작동 방식

#1 대소문자 통일

  • 소문자화(lowercasing): "Python" ↔ "python" 동일 취급
  • 검색·인덱싱 시 동일 규칙 적용

#2 공백·특수문자 정리

  • 연속 공백 하나로, 앞뒤 공백 제거
  • 구두점·특수문자 규칙 통일 (제거 또는 치환)

#3 유니코드·발음 구별 부호 정규화

  • NFD/NFC 등 정규형 통일 (예: é → e + accent 분리/결합)
  • 전각·반각 문자 통일 (한글·영문 혼용 환경)

#4 형태소·어간 통일 (선택)

  • Stemming: running → run, connected → connect
  • Lemmatization: better → good, am/are/is → be
  • 언어별 형태소 분석기 활용 (한국어: 형태소 분석기, 영어: NLTK, spaCy 등)

#3. 장점 (Benefits)

  • 표기 변이로 인한 검색 누락 감소 (recall 향상)
  • 다국어·다양한 입력에 대한 일관된 매칭
  • 인덱스 품질 향상으로 BM25·키워드 검색 성능 개선
  • 임베딩 전 적용 시 임베딩 공간 일관성에 도움

#4. 예제
